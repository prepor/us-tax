**Create `.github/workflows/refresh-tax-data.yml`** (per D-02, DATA-04).

The workflow must:

1. **Trigger** on:
   - `schedule: cron: "0 9 1 * *"` (1st of every month at 9:00 UTC)
   - `workflow_dispatch` (manual trigger button in GitHub UI)

2. **Permissions**: `contents: write` (needs to commit updated CSVs)

3. **Job steps**:
   a. Checkout repo (`actions/checkout@v4`)
   b. Setup Node 20 (`actions/setup-node@v4` with `cache: npm`)
   c. Install deps (`npm ci`)
   d. Run build-tax-data pipeline (`node scripts/build-tax-data.mjs`)
   e. Run full build (`npm run build`) to verify everything still compiles
   f. Check if any CSV files in `data/avalara/` changed using `git diff --quiet data/avalara/ || echo "changed=true" >> $GITHUB_OUTPUT`
   g. If changed: configure git user as `github-actions[bot]` / `github-actions[bot]@users.noreply.github.com`, add `data/avalara/`, commit with message `chore: refresh Avalara tax rate data`, push to main

4. **Fallback behavior** (per D-02): The workflow always runs `npm run build` which calls `build-tax-data.mjs`. If no new CSVs are downloaded (the download mechanism is a placeholder for now), the pipeline uses the committed CSVs. The workflow only commits if files actually changed.

5. **Note about future enhancement**: Add a comment block explaining that a future step can be added to download fresh CSVs from Avalara before the build step. For now, the workflow rebuilds from committed CSVs and serves as the infrastructure for when automated download is implemented.

The exact YAML content:

```yaml
name: Monthly Tax Data Refresh

on:
  schedule:
    - cron: "0 9 1 * *"  # 1st of every month at 9am UTC
  workflow_dispatch:        # Allow manual trigger

permissions:
  contents: write

jobs:
  refresh:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      # Future: add a step here to download fresh Avalara CSVs
      # e.g., curl or a Node script that fetches from Avalara's download page
      # For now, we rebuild from the committed CSV files.

      # Run the tax data pipeline
      - run: node scripts/build-tax-data.mjs

      # Verify full build still works
      - run: npm run build

      # Check if CSV source data changed
      - name: Check for changes
        id: changes
        run: |
          git diff --quiet data/avalara/ || echo "changed=true" >> $GITHUB_OUTPUT

      # Commit updated data if changed
      - name: Commit updated data
        if: steps.changes.outputs.changed == 'true'
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data/avalara/
          git commit -m "chore: refresh Avalara tax rate data"
          git push
```
