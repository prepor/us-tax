**2. resolveTaxJurisdiction()** internal function (per D-06):
```typescript
function resolveTaxJurisdiction(
  buyerState: string,
  buyerZip: string,
  sellerState?: string,
  sellerZip?: string,
): { rateState: string; rateZip: string; sourcingRule: "origin" | "destination" }
```
Logic:
- No seller info (undefined/empty) -> use buyer (destination). This is the default per D-04 (seller ZIP is optional).
- Interstate (sellerState !== buyerState) -> always destination (buyer's rates). Per D-06.
- Intrastate + origin-based state -> use seller location. Per D-06.
- Intrastate + destination-based state -> use buyer location.
- CA is simplified as origin-based (not hybrid). Per D-07 -- hybrid deferred to v2.

**3. applyProductTaxability()** internal function:
```typescript
function applyProductTaxability(
  category: ProductCategory,
  stateData: StateTaxData,
  subtotal: number,
): { taxableAmount: number; effectiveRate: number; explanation: string }
```
Logic:
- "general" -> fully taxable at state rate, return `{ taxableAmount: subtotal, effectiveRate: stateData.stateRate, explanation: "Taxable at state rate" }`
- For groceries/clothing/medicine/prepared_food -> look up `stateData.taxability[category]` rule
  - "exempt" -> `{ taxableAmount: 0, effectiveRate: 0, explanation: rule.note }`
  - "reduced" with `rule.rate` -> `{ taxableAmount: subtotal, effectiveRate: rule.rate, explanation: rule.note }`
  - "reduced" with `rule.threshold` + "exempt_below" -> if subtotal <= threshold: exempt, else: fully taxable at state rate
  - "reduced" with `rule.threshold` + "tax_excess" -> taxable amount is only the excess over threshold
  - "taxable" (default) -> `{ taxableAmount: subtotal, effectiveRate: stateData.stateRate, explanation: "Taxable at state rate" }`
- This mirrors the existing `applyTaxability()` in taxRules.ts but adapts for the new calculator's amount-based (not product-item) interface. NOTE: for threshold logic, the "price" is the full subtotal since the calculator takes a single amount (not per-unit pricing). The threshold comparison uses the subtotal directly.

**4. calculateFullTax()** -- the main export:
```typescript
export async function calculateFullTax(params: {
  buyerZip: string;
  sellerZip?: string;
  amount: number;
  category: ProductCategory;
}): Promise<TaxBreakdownResult>
```

Flow:
1. Resolve buyer state via `zipToState(buyerZip)`. If null, throw `new Error("Could not determine state for ZIP: {zip}")`.
2. If sellerZip provided and valid, resolve seller state via `zipToState(sellerZip)`.
3. Call `resolveTaxJurisdiction()` to determine which location's rates to use.
4. Call `loadStateData(rateState)` to fetch per-state JSON.
5. Call `lookupZip(stateData, rateZip)` to get the ZipEntry. If null (ZIP not in data), add a note "ZIP code {zip} not found in rate data for {state}; using state-level rate only" and create a synthetic ZipEntry with `{ region: stateName, combined: stateRate, county: 0, city: 0, special: 0 }`.
6. Apply product taxability via `applyProductTaxability()`. This gives the taxableAmount and an explanation.
7. Build JurisdictionBreakdown array. For each non-zero rate level:
   - State: `{ jurisdiction: "State", name: stateData.stateName, rate: stateData.stateRate, taxAmount: Math.round(taxableAmount * stateData.stateRate * 100) / 100 }`
   - County: `{ jurisdiction: "County", name: zipEntry.region, rate: zipEntry.county, taxAmount: Math.round(taxableAmount * zipEntry.county * 100) / 100 }` (only if county > 0)
   - City: same pattern for `zipEntry.city`
   - Special District: same pattern for `zipEntry.special`
   IMPORTANT per pitfall 3: compute totalTax from `Math.round(taxableAmount * zipEntry.combined * 100) / 100` (using the stored combined rate), NOT as sum of jurisdiction amounts. The jurisdiction amounts are informational.
   For taxability adjustments: if the category is exempt, all jurisdiction amounts should be 0. If reduced rate, the effective rate replaces the state rate portion; local rates may still apply (simplified: for reduced-rate categories, use `effectiveRate` for the combined calculation and prorate jurisdictions proportionally).
   SIMPLIFICATION for v1: When a category has reduced or exempt status, apply the taxability adjustment to the ENTIRE combined rate proportionally. That is:
   - If exempt: all jurisdiction amounts = 0, totalTax = 0
   - If reduced with a specific rate: `adjustmentFactor = effectiveRate / stateData.stateRate` (or 0 if stateRate is 0). Apply this factor to each jurisdiction amount. If the reduced rate is lower than the state rate, the local rates are also proportionally reduced. This is a v1 simplification -- real-world treatment varies by state.
   - If fully taxable: no adjustment.
8. Check `hasMultipleJurisdictions()` and set the flag. If true, add note "This ZIP code spans multiple tax jurisdictions. Showing the primary rate."
9. Add notes from taxability explanation.
10. If no seller ZIP was provided, add note: "Destination-based calculation (no seller location specified)."
11. If state has no sales tax (stateData.hasSalesTax === false), set all rates/amounts to 0 and add note "{stateName} does not impose a sales tax."

Return the complete `TaxBreakdownResult`.
